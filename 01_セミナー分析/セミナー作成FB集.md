# セミナー作成フィードバック集

> バックエンドセミナー構成作成時に出たFBをまとめたもの。今後のセミナー作成に活用する。

---

## 1. 全体構成に関するFB

### ノウハウ要素を充実させる
- セミナーには具体的なノウハウ要素を入れる
- 例：AIツールの紹介、AIの歴史・進化、業務効率化フレームワークなど
- 参加者が「今日だけで価値があった」と思える内容を提供する

### データと事例を豊富に
- 抽象的な話だけでなく、具体的なデータを入れる
- 市場規模、成長率、求人数など数字で示す
- 企業事例だけでなく、個人の成果事例も入れる
- 参加者と似た属性の事例があると刺さりやすい

### 時間配分の最適化
- 2時間（120分）に収まるように重複・無駄を省く
- 各STEPの時間配分を明確にする

---

## 2. 各STEPに関するFB

### オープニング（STEP1）
- 特典は具体的に提示する（例：プロンプト集）
- ルールはフロントセミナーと統一する
- 「期待値設定」は「スキルを無駄にしない」「実務に活かす」というニュアンスで
- 「最後まで聞く理由」は浅い表現を避け、具体的な価値を提示する

### 参加者との対話（STEP2）
- 自己紹介の前に入れる方が効果的
- 質問例：「BCでAIを使ってみて感じたこと」「今後AIで実現したいこと」
- 回答を拾ってリアクションし、仲間意識を醸成する

### 問題提起（STEP4）
- AI進化の年表は問題提起パートに入れる（危機感を煽る目的）
- ネット社会化との比較でAIの進化スピードを実感させる
- 正しい情報を使う（間違った情報はNG）
- BCの成果肯定→データへの橋渡しを丁寧に行う
- 「自分ごと化」の問いかけを入れる
  - 例：「あなたの会社でAIを使いこなしている人は何人？」
  - 例：「1年後、あなたの仕事はどうなっていると思いますか？」

### 解決策の提示（STEP5）
- ステップ数は3つが適切（数字的に覚えやすい）
- フロント商品（BC）と今日のセミナーでSTEP1は達成済みという見せ方
- 「フロント商品はゴールではなくスタート」というメッセージを強調
- AI人材レベルはLv.1〜Lv.5の5段階で表現（最高がLv.4だと微妙）

### ノウハウ解説（STEP6-8）
- AIツールカテゴリマップで「知らないツールいくつありますか？」と問いかける
- フロント商品（BC）のカリキュラム内容と連動させる
- DAY1〜DAY3で学んだことを振り返り、他の業務への応用を示す

### 社会的証明（STEP9）
- 前のSTEPからの橋渡しを入れる
- 初回リリースの場合、「受講生の事例」という見せ方はできない
- 企業導入事例＋個人の成果事例の両方を入れる
- 「特別な人ではない、普通の人でもできた」という共通点を強調

### 教育パート（STEP10）
- 反論を「点」ではなく「線」で教育する
- 複数の反論を別々に処理するのではなく、一貫したストーリーで教育
- 教育の順番：「環境の必要性」→「タイミング」
  - まず「一人で学ぶ限界」を教育（情報過多、取捨選択の難しさ、迷って止まる）
  - 次に「環境で学ぶ価値」を教育（すぐ聞ける、最新情報が届く、仲間がいる）
  - 最後に「今このタイミング」を教育（フロント商品の感覚が残っているうちに）
- 得られる未来の話を入れて、次のSTEPへ橋渡しする

### オファー・クロージング（STEP11）
- サブスクをメインで訴求、事業者向けにはコンサルプランも紹介
- サブスクとコンサルを別々ではなく、線で説明する（コンサルはサブスクの上位版）
- 講師のTアップをゴリゴリ推す
  - キャッチアップ力（毎日情報収集、取捨選択できる）
  - 企業への導入実績（大手企業での研修、具体的な削減時間）
  - 業界知識×AI知識の掛け合わせ
- 個別相談の具体的な価値を明確にする
  - 例：「30分であなた専用のAI活用ロードマップをお渡しします」
  - 例：「業務の棚卸し」「どこをAI化すべきか診断」
- 限定性は具体的な数字で（「先着〇名」ではなく「先着10名」）

## 3. 論理構成に関するFB

### 感情の流れを設計する
- 各STEPで参加者の感情がどう変化するかを明確にする
- Before→Afterで感情変化を書き出す
- 最終ゴール（望む感情状態）から逆算して設計する

### 橋渡しを丁寧に
- STEP間の接続を丁寧に行う
- 唐突に次の話題に移らない
- 「ここまでは〇〇の話でした。では次に〇〇について...」という流れ

### 伏線を張る
- 後で回収する内容は、事前に伏線を張っておく
- 例：「これだけのツールを一人で試すのは無理」→「だから環境が必要」

### 一貫した軸をズラさない
- 最初に提示した問題と最後の解決策が一直線
- 途中で新しい問題を追加しない

## 4. 売れるセミナーにするためのFB

### 「自分ごと化」させる
- 社会全体の話だけでなく、「あなた自身の話」にする
- 問いかけで参加者に考えさせる
- 「あなたの会社では？」「あなたの仕事では？」

### 個別相談に申し込む「明確な理由」を作る
- 「まずは話を聞いてみて」だと弱い
- 個別相談自体が価値あるもの（無料コンサル）として見せる
- 「これだけでも今日参加した価値がある」と言い切る

### フロント商品からバックエンドへの接続
- 「フロント商品で扉を開けた、バックエンドで扉の向こうに進む」
- フロント商品の感覚が残っているうちに次に進ませる
- 「フロント商品はゴールではなく、スタート」を強調

### 商品は「環境」として見せる
- 「これを学べます」ではなく「ここに来れば変われる」
- サポート体制、コミュニティ、伴走を強調

### 最後に売るのは「未来」
- 商品説明より未来の話を多くする
- 機能より「変化後の生活」を描写する

---

## 5. 正確性に関するFB

### データは正確に
- AI市場規模、成長率などは最新の正確なデータを使う
- 間違った情報を入れるのはNG
- 不明な場合はウェブ検索で確認する

### AIの歴史・年表は正確に
- ChatGPT、GPT-4、Claude、Geminiなどのリリース日を正確に
- 「〇〇ができるようになった」という進化の内容も正確に

## 6. チェックリスト

セミナー構成完成時に確認する項目：

### 構成チェック
- [ ] ノウハウ要素が十分に入っているか
- [ ] データと事例が豊富か
- [ ] 時間配分は適切か
- [ ] 各STEP間の橋渡しがあるか
- [ ] 参加者の感情変化が設計されているか

### 内容チェック
- [ ] 「自分ごと化」の問いかけがあるか
- [ ] フロント商品との接続が明確か
- [ ] 教育パートが「線」になっているか
- [ ] 講師のTアップが十分か
- [ ] 個別相談の具体的な価値が明確か
- [ ] 限定性が具体的な数字か

### 正確性チェック
- [ ] データは正確か（最新情報か）
- [ ] AIの歴史・年表は正確か
- [ ] 間違った情報が含まれていないか

### 売れるセミナー10原則チェック
- [ ] ①セミナーの役割は「情報」ではなく「意思決定」
- [ ] ②最初に「今の自分」を言語化させる
- [ ] ③否定しない。構造で限界を見せる
- [ ] ④解決策は段階的に見せる
- [ ] ⑤全体像を途中で見せる
- [ ] ⑥危機感は煽らず想像させる
- [ ] ⑦行動しない理由は先回りして潰す
- [ ] ⑧商品は「環境」として見せる
- [ ] ⑨最後に売るのは「未来」
- [ ] ⑩一貫した軸をズラさない
